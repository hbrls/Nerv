# TASK-002 · Pilot 收束任务模板

> WORKFLOW：`.agents/workflows/pilot/WORKFLOW.md`
> 主题：`{{TOPIC}}`

## 背景

本任务由 pilot 创建，用于在上一轮 LENS 产出完成后进行收束判断。

本模板编号表示相对顺序。落地时，若当前 pilot TASK 编号为 `NNN`，本模板应生成当前 Vision 目录内的 `TASK-(NNN+2).md`，输出写入当前 Vision 目录内的 `C-(NNN+2).md`。

## 输入

读取以下文件或文件集合：

- `{{CURRENT_TASK_PATH}}`
- `{{C_OUTPUTS_GLOB}}`

## 判断

1. 当前 TASK 指定的输入是否足够支持收束判断？
2. 已有 `C-*` 产出是否充分回应当前目标？
3. 当前讨论是否已经形成足够清晰、可执行、可交付的结论？
4. 如果需要继续，下一轮最有价值的 TASK 应该调用哪个 LENS、补齐哪个任务契约内缺口、产出什么文件化增量？

## 约束

- 只读取本任务输入中列出的文件
- 不读取其它 `TASK-*`
- 不读取或写入其他 Vision Id 目录
- 不把当前 LENS 列为输入
- 不执行非本任务要求的代码或业务文件变更
- 除本任务输出、当前 Vision 目录内的下一轮 TASK（继续时含 `TASK-(NNN+1)` 和 `TASK-(NNN+2)` 两份；结束时含 `TASK-(NNN+1)` 一份，基于 TASK-003 模板）外，不创建、修改或删除其他文件
- 不替下一轮 LENS 设计研究方法、定义执行优先级、设定隐性评价标准或预设结论
- 不把未被当前规则块或任务契约授权的新规则写成单个后续 TASK 的特别指令；如规则不足，只能在本轮输出中建议更新任务契约或规则源
- 假设登记只记录影响本轮收束判断的未定项，不得登记给未来 TASK 设定的执行预期

## 输出

将简洁收束判断写入 `{{OUTPUT_C_PATH}}`，只保留判断方向、关键依据、缺口或可结束理由，以及已落成的调度结果；不要复述各 `C-*` 的完整内容，不替 LENS 展开分析，也不替 checkpoint 写总结。

如果判断讨论应该结束，基于 TASK-003 模板创建一份 `TASK-(NNN+1).md`，该任务指定调用 checkpoint 工作流（`.agents/workflows/checkpoint/WORKFLOW.md`）。不要同时创建 TASK-(NNN+2).md。

如果判断讨论仍需继续，必须同时创建两份 TASK：
- 当前 Vision 目录内的 `TASK-(NNN+1).md`：选定的 LENS 任务，在其中明确调用的 LENS、输入文件、任务契约内缺口、期望文件化增量、输出路径
- 当前 Vision 目录内的 `TASK-(NNN+2).md`：配套的 pilot 收束任务，用于在该 LENS 完成后进行收束
