## 4. 核心问题（必须回答才能动）

1. **"读取"的对象是什么？** blames / 仓库源码 / MR 变更 / Issue / Pipeline 日志 / 都不是。
2. **读给谁用？** 直接展示给人看，还是喂给 backstage.js 的某个能力（如 catalog 同步、techdocs、search）。
3. **与 Gitea 的关系？** 替换、并存（按项目类型路由）、还是分层（Gitlab 为主、Gitea 兜底）。
4. **成功标准？** 至少包含：(a) 能拉到数据；(b) 拉到后能用于既定的下游能力；(c) 性能 / 限频 / 鉴权上能扛得住。
5. **依赖与约束？** 需要 Gitlab token / self-hosted vs SaaS / 网络可达性 / API 版本。
6. **优先级？** 是否阻塞 VISION-001 的最终合成（Index.md 暗示会，但不强制）。
